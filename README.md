# 🧵 AtelierDB — База данных для ателье по пошиву одежды

Реляционная база данных для автоматизации работы ателье: учёт клиентов, заказов, материалов, поставщиков и продаж готовой продукции.

**СУБД:** Microsoft SQL Server · **Язык:** T-SQL · **Среда:** SQL Server Management Studio

## Задача

Вариант 36 — «Информационная поддержка работы ателье».
Ателье осуществляет пошив одежды под заказ и продаёт готовые изделия. Покупатели — физические и юридические лица. БД учитывает поставки материалов и ведёт учёт поставщиков.

## Структура БД

**14 таблиц**, нормализация до **3НФ**, 13 внешних ключей.

| Группа | Таблицы |
|--------|---------|
| Справочники | `ClientTypes`, `Positions`, `Services`, `Materials`, `Suppliers`, `FinishedProducts` |
| Основные сущности | `Clients`, `Employees`, `CustomOrders`, `SupplyDeliveries` |
| Связующие | `OrderServices`, `OrderMaterials`, `SupplyDetails`, `ProductSales` |

## Реализованные объекты

| Тип | Кол-во | Примеры |
|-----|--------|---------|
| Представления (View) | 3 | Остатки материалов, список сотрудников, редактируемое представление клиентов |
| Скалярные функции | 2 | Количество заказов клиента, общая сумма продаж |
| Табличная функция | 1 | Материалы с низким остатком (параметризуемый порог) |
| Хранимые процедуры | 3 | Создание заказа, обновление цены, безопасное удаление клиента |
| Триггеры | 3 | Автопополнение склада при поставке, автосписание при расходе, защита от удаления должности |

## Ключевые особенности

- **Автоматизация склада** — триггеры автоматически обновляют остатки материалов при поставках и списаниях
- **Защита данных** — процедура безопасного удаления клиента (проверка зависимостей), триггер защиты справочника должностей
- **Аналитические запросы** — выручка по периодам, топ-клиенты, загрузка сотрудников, маржинальные услуги
